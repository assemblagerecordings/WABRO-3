1. SHORT VERSION

WABRO X ASSEMBLAGE is my attempt to build a living creative operating system for electronic music.

It starts from my own real music projects, especially neurofunk / drum and bass work made in Bitwig and Ableton. The goal is not to ask AI to generate generic music from scratch. The goal is to make an AI-assisted system that understands my existing tracks, snares, basses, plugin chains, project saves, stems, research, visuals, and performance ideas, then helps mutate and extend them.

The central idea is:

Track = DNA
AI = Mutation Engine
EEG / brain-state = live modulation source
Bitwig + visuals = responsive body
Archive = memory
Platform = shared mutation field

The first practical milestone is to make one track become a playable living instrument:

Backwards V5 + Muse 2 EEG or simulator + 8 Bitwig macro controls + snare geometry visualiser + persistent state/logs.

The long-term ambition is to create a new kind of open music platform where songs are not just audio files. They are living systems with stems, project context, visual worlds, state data, remix/mutation logic, and attribution.

But I do not want to jump straight to the platform. The correct order is:

5. THE FIRST PLAYABLE SYSTEM

The current practical build is:

Backwards V5
+ Muse 2 EEG headset or simulator
+ 8 macro controls
+ WABRO snare geometry
+ persistent state and logs

Backwards V5 is one of my Bitwig projects and is the first target for mutation.

The state engine should produce states such as:

- Signal
- Pressure
- Mutation

The state engine should not directly control random plugin parameters. It should convert messy input data into stable, playable macro values.

Possible inputs:

- alpha
- beta
- gamma
- theta
- delta
- blink
- jaw clench
- motion or other sensor data if available

Important: I do not want to overclaim that EEG directly detects emotions. It is better to call this brain-state, affective-state, pressure-state, or performance-state mapping.


6. THE FIRST 8 MACROS

The first version should use 8 macro controls:

1. Snare Fundamental
2. Snare Noise
3. Snare Transient Bite
4. Bass Filter Motion
5. Bass Distortion
6. Spectral Rotation
7. Space / Width
8. Glitch / Fill Probability

These macros should control large musical ideas, not every tiny parameter.

For example:

- more pressure can increase bass distortion, transient bite, or density
- more signal/coherence can open space or stabilise the geometry
- mutation can increase spectral rotation or fill probability
- collapse can reduce clarity, narrow space, or deform the visual system

The goal is a playable system, not chaos.
